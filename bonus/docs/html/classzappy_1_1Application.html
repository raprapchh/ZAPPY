Detailed Description

Main application class that manages the Zappy GUI client.

The Application class serves as the central coordinator for the entire GUI client. It manages the game loop, handles events, coordinates rendering, manages the network connection to the server, and maintains the world state.

The application follows a typical game engine architecture with:

  • Event-driven input handling
  • Fixed timestep updates for game logic
  • Separate render loop for visual output
  • Modular systems (Network, World, Renderer, UI)

Key responsibilities:

  • Initialize and manage all subsystems
  • Run the main game loop (events, update, render)
  • Handle window management and user input
  • Process network messages from the Zappy server
  • Coordinate between different rendering modes (2D/3D)
See also
IRenderer for rendering abstraction
World for game state management
NetworkClient for server communication
