Detailed Description

Static class providing movement and navigation services for Zappy AI.

This class implements all movement-related functionality including basic
locomotion, environmental scanning, and intelligent navigation. The movement
system is designed around the Zappy vision model where players can see
tiles in their field of view and navigate to specific positions.

Vision System Layout:
The Look command returns tiles in a specific pattern based on player level:
- Level 1+: 3x3 grid around player (9 tiles, indices 0-8)
- Higher levels: Extended vision with additional rows

Tile indexing follows this pattern:
    4 5 6
    1 0 2  <- Player at index 0
    7 8 9

Navigation capabilities scale with player level, allowing access to
more distant tiles as vision range increases.

List[str] movement.MovementManager.look_around ( Player player)
static 
Get list of visible tiles and their contents around the player.

Executes the Look command and parses the response into a list of
tile contents. Each tile string contains space-separated resource
names present on that tile. The vision range and tile count depend
on the player's current level.

Tile contents format:
- Empty tiles: empty string or whitespace
- Resource tiles: space-separated resource names (e.g., "food linemate")
- Multiple resources: all resources listed in single string

Error handling:
- Returns empty list on communication errors
- Handles malformed server responses gracefully
- Robust parsing for various response formats

Args:
    player (Player): Player to get vision information for
    
Returns:
    List[str]: List of tile contents strings, indexed by tile position
              Empty list if Look command fails

◆ navigate_to_tile()

movement.MovementManager.navigate_to_tile ( Player player,
int tile_index )
static 
Navigate player to specific tile within vision range.

Implements intelligent pathfinding to reach a target tile identified
by its index in the vision system. The method uses the tile layout
to determine optimal movement sequences for reaching the destination.

Navigation algorithm:
- Index 0: Player's current position (no movement needed)
- Index 1-3: Adjacent tiles requiring simple movements
- Index 4-8: Distant tiles requiring multi-step navigation
- Level-aware: Higher level players can reach more distant tiles

Movement patterns by tile index:
- Index 1: Left + Forward (left adjacent)
- Index 2: Forward (front adjacent)
- Index 3: Right + Forward (right adjacent)
- Index 4-8: Complex multi-step sequences for distant tiles

Level restrictions:
- Tiles beyond basic vision (index > 3) require level 2+
- Navigation complexity scales with vision range

Args:
    player (Player): Player to navigate
    tile_index (int): Target tile index from vision system (0-8+)
