Detailed Description

Abstract interface for rendering systems in the Zappy GUI.

The IRenderer interface defines the contract that all rendering implementations must follow. This allows the application to support multiple rendering backends (2D SFML, 3D Raylib, etc.) through a common interface.

The interface supports:

  • Multiple rendering modes (2D, 3D, isometric)
  • Quality settings and visual options
  • Debug visualization modes
  • Event handling for renderer-specific inputs
  • Window management and lifecycle
  • Animation triggers for game events

Implementations should handle:

  • Efficient rendering of large worlds
  • View frustum culling for performance
  • Level-of-detail (LOD) rendering
  • Smooth animations and transitions
  • UI overlay rendering
See also
Renderer for 2D SFML implementation
Renderer3D for 3D Raylib implementation
