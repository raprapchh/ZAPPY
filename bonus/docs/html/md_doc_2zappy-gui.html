Architecture du Client GUI Zappy

Vue d'ensemble

Le client GUI est responsable de la visualisation en temps réel du monde Zappy. Il se connecte au serveur en tant que client spécial "GRAPHIC" et reçoit des mises à jour sur l'état complet du jeu.

Optimisations de rendu

1. Culling

// Ne rendre que ce qui est visible
sf::FloatRect viewBounds = camera.getViewBounds();
for (const auto& entity : world.getEntities()) {
if (viewBounds.intersects(entity->getBounds())) {
renderEntity(entity);
}
}

2. Batching

// Grouper les draw calls par texture
std::map<sf::Texture*, std::vector<sf::Sprite>> batches;
for (const auto& sprite : sprites) {
batches[sprite.getTexture()].push_back(sprite);
}

Système de réseau

Thread de réseau séparé

class NetworkThread {
private:
std::thread thread_;
std::atomic<bool> running_;
// Communication thread-safe
ThreadSafeQueue<Message> incoming_messages_;
ThreadSafeQueue<Command> outgoing_commands_;
void run() {
while (running_) {
// Receive
if (socket_.hasData()) {
auto msg = parseMessage(socket_.receive());
incoming_messages_.push(msg);
}
// Send
if (!outgoing_commands_.empty()) {
socket_.send(outgoing_commands_.pop());
}
}
}
};

Parsing des messages

class MessageParser {
private:
std::unordered_map<std::string, MessageHandler> handlers_;
public:
void registerHandlers() {
handlers_["msz"] = &MessageParser::handleMapSize;
handlers_["bct"] = &MessageParser::handleTileContent;
handlers_["pnw"] = &MessageParser::handlePlayerNew;
// ... etc
}
Message parse(const std::string& raw) {
auto tokens = tokenize(raw);
if (handlers_.count(tokens[0])) {
return handlers_[tokens[0]](tokens);
}
return Message{MessageType::UNKNOWN};
}
};

// Position actuelle vs position serveur

if (server_pos_ != interpolated_pos_) {
interpolated_pos_ = lerp(interpolated_pos_, server_pos_, dt * 5.0f);
}
}

Points d'attention

1. Thread safety

  • Queue thread-safe pour messages
  • Pas de modification du world depuis le render thread
  • Synchronisation avec mutex minimal

2. Memory management

  • Smart pointers pour tout
  • Éviter les copies inutiles
  • Pool d'objets pour les particules

3. Scalabilité

  • Supporter des cartes jusqu'à 100x100
  • Gérer 100+ joueurs simultanés
  • Maintenir 60 FPS
