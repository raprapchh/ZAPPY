Detailed Description

Main AI controller class for Zappy game client.

This class orchestrates all AI decision-making processes and serves as the
central coordinator between different specialized managers. It implements
a priority-based decision system where survival needs take precedence over
progression goals.

The AI operates in distinct phases:
1. Connection establishment and initial setup
2. Survival-focused resource gathering
3. Level-specific strategic progression
4. Team coordination and communication
5. Adaptive behavior based on game state

Attributes:
    config (GameConfig): Game connection and configuration parameters
    player (Player): Current player instance with state information
    last_food_check (int): Timestamp of last food level verification

Execute strategy for levels 3 and above with leadership focus.

Higher level strategy shifts focus from individual progression
to team leadership and coordination. Players at these levels:

1. Have more complex resource requirements needing team coordination
2. Should act as leaders guiding lower-level players
3. Focus on exploration and map knowledge sharing
4. Coordinate elevation attempts requiring multiple players

The strategy emphasizes:
- Survival maintenance without aggressive resource competition
- Leadership broadcasting to organize team efforts
- Exploration to discover resource-rich areas
- Strategic positioning for team elevation attempts

Design rationale:
- Higher levels require team coordination, not individual effort
- Leadership communication helps organize team progression
- Exploration provides valuable intelligence for team strategy

Execute level 1 specific strategy focusing on basic survival and first elevation.

Level 1 strategy is the foundational behavior that focuses on:
1. Ensuring basic survival with adequate food reserves
2. Collecting linemate (the only resource required for level 2)
3. Attempting elevation when resources are sufficient

The strategy uses a simple state machine approach:
- COLLECTING state: Actively searching for linemate
- ELEVATING state: Attempting elevation ritual

Design decisions:
- Survival always takes precedence over resource collection
- Only one linemate needed, making strategy straightforward
- State transitions are clear and deterministic

◆ run_intelligent_strategy()

zappy_ai.ZappyAI.run_intelligent_strategy ( self) 
Execute the main AI strategy loop with adaptive decision-making.

This is the core method that implements the AI's main game loop.
It uses a priority-based decision system that continuously evaluates
the player's state and executes appropriate strategies based on:

1. Critical survival needs (food < 2)
2. Current player level and progression requirements  
3. Team coordination opportunities
4. Resource collection and exploration
5. Dynamic timing adjustments based on food security

The method implements several key algorithms:
- Adaptive sleep timing based on food levels
- Periodic food level monitoring with configurable intervals
- Level-specific strategy delegation
- Broadcast coordination with anti-spam measures
- Graceful interruption handling for clean shutdown

Raises:
    KeyboardInterrupt: Propagated from user interruption for clean shutdown
